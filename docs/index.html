The purpose of this repository is to establish standardized templates that will be utilized in all development processes going forward.

Getting Started with DOCS

When organizing the "docs" directory, it is important to adhere to the same structure as the development process. Please exclude any files that do not require documentation.

After creating the similar structure, you can map the corresponding .py file to the .md file using the command ::: folder_name.folder_name.file_name (without the extension). For instance, ::: template_project.data_sync.admin would serve as an example.

If any code files are deleted, it is important to update the corresponding documentation in the docs/ directory as well, especially if you intend to host the documentation for that file.

Please ensure that you make changes in the "mkdocs.yml" file and include all the newly added or deleted files

WRITING DOCS:

Follow the conventions of editing .md file to edit.

COMMANDS:

  • mkdocs serve - Start the live-reloading docs server. Running the command will host the docs server on localhost. Use this while devloping, as the changes will be reflected immediately.
  • mkdocs build - Build the documentation site.

DEPLOYING DOCS:

1) mkdocs gh-deploy - Deploy your documentation to GitHub Pages
